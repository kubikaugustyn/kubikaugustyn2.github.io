    Validace přezdívky

    Seznam mnou extrahovaných možných slov v přezdívce je k nalezení zde.

    Server (nejspíše) validuje přezdívky přibližně takto:

    1. (převede přezdívku na malá písmena)
    2. Obsahuje přezdívka uživatelské jméno?
    3. Obsahuje přezdívka dvě části oddělené podtržítkem?
    4. (rozělí přezdívku na dvě části)
    5. Jsou části v seznamu povolených slov (viz. seznam mnou extrahovaných možných slov)?

    Tato validace by se ale dala vylepšit o tuto kontrolu: "Je velké písmeno jen na začátku a nikde jinde?".

    Také bych vylepšil výběr přezdívky, řekněme že jsme malé dítě, které si s rodičem vybírá přezdívku. Klikám přes možné přezdívky až narazíme na "Liberijský_poník". Mě se líbí poník, ale chtěl bych přezdívku "Modrý_poník", jenže si v aktuálním UI nemůžu nechat poníka a změnit si první slovo na "Modrý", jenže musím klikat dál a věřit náhodě, šance na přezdívku "Modrý_poník" je velmi malá, tedy jako malé dítě na modrého poníka mohu zapomenout. Tento problém by se dal vyřešit s pomocí dvou tzv. dropdown menu, ve kterých by se dala i hledat slova do přezdívky a dala by se samostatně volit slova přezdívky.

    AMF zprávy nejsou brány kompletně

    AMF zprávy jsou reprezentovány na straně serveru Java třídou "flex.messaging.io.amf.ActionMessage"(?) (kód lze nalézt např. zde). Tyto zprávy mimo jiné podporují seznam "bodies", který slouží k možnosti odeslání více "requestů/zpráv" v rámci jednoho HTTP requestu.

    Když například získáváme přístupový token pro AMF API, pošleme tento POST https://decko.ceskatelevize.cz/rest/Token/amfJson request:

    {
  "version": 3,
  "headers": [],
  "bodies": [
    {
      "targetURI": "createToken",
      "responseURI": "/1",
      "data": [
        {
          "appUuid": "...",
          "posterEnabled": "false",
          "closeButtonEnabled": "false",
          "controllerEnabled": "false"
        }
      ]
    }
  ]
}

    A dostaneme odpověď 200 OK:

    {
  "version": 3,
  "headers": [],
  "bodies": [
    {
      "targetURI": "/1/onResult",
      "responseURI": "",
      "data": [
        {
          "flashvars": ...,
          "integrativeContainer": ...
        }
      ],
      "replyMethod": "onResult"
    }
  ],
  "headerCount": 0, // Mimochodem, toto není součást ActionMessage
  "bodyCount": 1
}

    Pokud ale pošleme stejný request, ale s více "AMF requesty" v poli "bodies", jako např. tento:

    {
  "version": 3,
  "headers": [],
  "bodies": [
    {
      "targetURI": "createToken",
      "responseURI": "/1",
      "data": ...
    },
    {
      "targetURI": "createToken",
      "responseURI": "/2",
      "data": ...
    }
  ]
}

    Dostaneme odpověď 404 Not Found, místo (v tomto případě) očekávaných dvou tokenů (ano, je to trochu nerealistický příklad, ale server by měl přijímat obě dvě položky pole "bodies").

    Změna hodnot v paměti zkompilovaných Unity Déčko her na webu Déčka

    Pokud získáme refereci na paměť WebAssembly programu (dále jen hry) při pozastavení JavaScriptu s pomocí tzv. devtools a uložíme ji do proměnné mem:

    window.mem = memories.$memory // Pokud pozastavíte ve WASM kódu

window.mem = Module.asm.memory // Pokud pozastavíte podpůrný JS soubor, jako webgl.framework.js(.br)

    Poté můžeme využít tento skript pro nalezení adresy skóre v paměti hry:

    let addresses = null
let access = null
const begin = score => {
    addresses = []
    access = new Int32Array(mem.buffer)

    for (let i = 0; i < access.length; i++)
        if (access[i] === score) addresses.push(i)
}
const update = score => {
    addresses = addresses.filter(i => access[i] === score)
}
const changeScore = newScore => {
    addresses.forEach(i => {
        access[i] = newScore
    })
}

    Použití

    1. Začněte novou hru
    2. Pozastavte hru a vyvolejte begin(<aktuální skóre>), addresses.length, poté hrajte dál.
    3. Pozastavte hru a vyvolejte update(<aktuální skóre>), addresses.length, poté hrajte dál dokud má seznam addresses moc adres (většinou více než jednu).
    4. Nyní máte adresu skóre v paměti hry a můžete zavolat changeScore(1_234_567_890) pro změnu skóre na jakoukoliv hodnotu.
    5. Poté skončete hru (nechytněte toaleťák...), a vaše vysoké skóre bude odesláno na server.
